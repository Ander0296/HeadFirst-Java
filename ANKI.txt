¿Qué camino recorre un programa Java desde el archivo fuente hasta ejecutarse?	Se escribe el código en un archivo .java, el compilador javac lo traduce a bytecode (archivo .class), y la JVM (máquina virtual de Java) ejecuta ese bytecode. El bytecode es el mismo para todos los sistemas; lo que cambia es la JVM de cada plataforma.
¿Por qué Java cumple "write once, run anywhere" (escribí una vez, corré en cualquier parte)?	Porque el compilador no genera código de máquina para un procesador concreto sino bytecode neutro, y cada sistema operativo tiene su propia JVM que sabe ejecutar ese mismo bytecode sin recompilar el programa.
¿Cuál es la estructura mínima de una clase ejecutable en Java?	public class MiClase { public static void main(String[] args) { } } — todo el código vive dentro de una clase, la ejecución arranca en main, y el archivo debe llamarse igual que la clase pública: MiClase.java.
¿Por qué el método main es obligatorio y cuál es su firma exacta?	Es el punto de entrada: la JVM busca exactamente public static void main(String[] args) y ejecuta sus sentencias en orden. Sin main, la clase compila pero no se puede ejecutar como programa.
¿Cuáles son las tres cosas que se hacen dentro de un método en Java?	Statements (sentencias que hacen algo y terminan en punto y coma), loops (bucles como while que repiten un bloque), y branching (ramificaciones con if/else que eligen un camino según una condición).
¿Cómo funciona un bucle while en Java?	while (condición) { } repite el bloque mientras la condición boolean sea true. La condición se evalúa ANTES de cada vuelta: si ya empieza en false, el bloque no se ejecuta ni una vez.
¿Cómo funciona if / else en Java y de qué tipo debe ser la condición?	if (condición) { } ejecuta su bloque solo si la condición es true; else { } captura el caso contrario. La condición debe ser un boolean (en Java no vale un número como en otros lenguajes) y va siempre entre paréntesis.
¿Cómo se declara un array en Java y cómo se accede a sus elementos?	int[] numeros = new int[4]; crea un array de 4 posiciones. Se accede con corchetes empezando en cero: numeros[0] es el primero y numeros[numeros.length - 1] el último. Salirse del rango lanza una excepción en ejecución.
¿Cómo se genera un número entero al azar con java.util.Random?	Se importa con import java.util.Random; se crea el generador con Random r = new Random(); y r.nextInt(5) devuelve un entero entre 0 y 4: el límite que le pasás es exclusivo.
¿Qué diferencia hay entre System.out.println() y System.out.print()?	Ambos imprimen en consola, pero println() agrega un salto de línea al final y print() deja el cursor en la misma línea, así lo siguiente que imprimas continúa a continuación.
¿Cuál es el ciclo de trabajo para probar un programa Java desde la terminal?	Guardar el archivo fuente (MiClase.java), compilar con javac MiClase.java (genera MiClase.class con el bytecode), y ejecutar con java MiClase — sin extensión, porque le pasás el nombre de la clase, no del archivo.
¿Qué dos reglas estructurales tiene que cumplir un archivo Java para compilar, más allá de que la sintaxis interna esté bien?	Todo método tiene que estar declarado dentro de una clase, y toda instrucción, bucle o if tiene que estar dentro de un método: ninguno de los dos puede aparecer suelto ni ir directo en el archivo o en el cuerpo de la clase.
¿Qué pasa si escribís un método como public static void main sin encerrarlo dentro de una clase?	El archivo no compila. En Java no existen métodos ni funciones sueltas: todo método tiene que vivir dentro de la declaración de una clase, con la forma class NombreClase { }.
¿Qué pasa si escribís un while o un if directamente en el cuerpo de una clase, sin ponerlo dentro de un método?	Tampoco compila. El cuerpo de una clase solo admite declaraciones (variables, métodos, otras clases); las instrucciones ejecutables como while o if necesitan estar dentro del cuerpo de un método.
¿Cómo se traza (rastrea) a mano un bucle while con varias variables para predecir qué imprime un programa?	Se anota el valor de cada variable justo antes de entrar al bucle, y se lo actualiza línea por línea en cada vuelta, aplicando cada instrucción y cada if en el orden exacto del código — nunca se adivina el resultado sin pasar por cada vuelta una por una.
¿Qué imprime System.out.print(x + "" + y + " ") si x vale 4 e y vale 10 (ambos int)?	Imprime el texto "410 ": los enteros se convierten a texto y se pegan uno al lado del otro sin espacio entre ellos (el espacio final es literal, viene del " " del final). No es una suma: 4 + 10 daría 14, un resultado totalmente distinto.
¿Qué imprime System.out.println(1 + 2 + "3") en Java, y por qué no da lo mismo que System.out.println("1" + 2 + 3)?	El primero imprime "33": como 1 y 2 son ambos int, Java los suma primero (da 3) y recién después concatena con el String "3". El segundo imprime "123": al arrancar con un String, todos los + que siguen concatenan como texto en vez de sumar. El orden de los operandos cambia todo.
¿Por qué conviene usar print() en vez de println() cuando querés armar una palabra o frase juntando varios fragmentos de texto?	Porque print() no salta de línea: cada System.out.print() sucesivo se pega justo después del anterior, en la misma línea. Con println() cada fragmento quedaría en su propia línea, rompiendo la palabra que se quiere armar.
