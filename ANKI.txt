¿Qué camino recorre un programa Java desde el archivo fuente hasta ejecutarse?	Se escribe el código en un archivo .java, el compilador javac lo traduce a bytecode (archivo .class), y la JVM (máquina virtual de Java) ejecuta ese bytecode. El bytecode es el mismo para todos los sistemas; lo que cambia es la JVM de cada plataforma.
¿Por qué Java cumple "write once, run anywhere" (escribí una vez, corré en cualquier parte)?	Porque el compilador no genera código de máquina para un procesador concreto sino bytecode neutro, y cada sistema operativo tiene su propia JVM que sabe ejecutar ese mismo bytecode sin recompilar el programa.
¿Cuál es la estructura mínima de una clase ejecutable en Java?	public class MiClase { public static void main(String[] args) { } } — todo el código vive dentro de una clase, la ejecución arranca en main, y el archivo debe llamarse igual que la clase pública: MiClase.java.
¿Por qué el método main es obligatorio y cuál es su firma exacta?	Es el punto de entrada: la JVM busca exactamente public static void main(String[] args) y ejecuta sus sentencias en orden. Sin main, la clase compila pero no se puede ejecutar como programa.
¿Cuáles son las tres cosas que se hacen dentro de un método en Java?	Statements (sentencias que hacen algo y terminan en punto y coma), loops (bucles como while que repiten un bloque), y branching (ramificaciones con if/else que eligen un camino según una condición).
¿Cómo funciona un bucle while en Java?	while (condición) { } repite el bloque mientras la condición boolean sea true. La condición se evalúa ANTES de cada vuelta: si ya empieza en false, el bloque no se ejecuta ni una vez.
¿Cómo funciona if / else en Java y de qué tipo debe ser la condición?	if (condición) { } ejecuta su bloque solo si la condición es true; else { } captura el caso contrario. La condición debe ser un boolean (en Java no vale un número como en otros lenguajes) y va siempre entre paréntesis.
¿Cómo se declara un array en Java y cómo se accede a sus elementos?	int[] numeros = new int[4]; crea un array de 4 posiciones. Se accede con corchetes empezando en cero: numeros[0] es el primero y numeros[numeros.length - 1] el último. Salirse del rango lanza una excepción en ejecución.
¿Cómo se genera un número entero al azar con java.util.Random?	Se importa con import java.util.Random; se crea el generador con Random r = new Random(); y r.nextInt(5) devuelve un entero entre 0 y 4: el límite que le pasás es exclusivo.
¿Qué diferencia hay entre System.out.println() y System.out.print()?	Ambos imprimen en consola, pero println() agrega un salto de línea al final y print() deja el cursor en la misma línea, así lo siguiente que imprimas continúa a continuación.
¿Cuál es el ciclo de trabajo para probar un programa Java desde la terminal?	Guardar el archivo fuente (MiClase.java), compilar con javac MiClase.java (genera MiClase.class con el bytecode), y ejecutar con java MiClase — sin extensión, porque le pasás el nombre de la clase, no del archivo.
¿Qué dos reglas estructurales tiene que cumplir un archivo Java para compilar, más allá de que la sintaxis interna esté bien?	Todo método tiene que estar declarado dentro de una clase, y toda instrucción, bucle o if tiene que estar dentro de un método: ninguno de los dos puede aparecer suelto ni ir directo en el archivo o en el cuerpo de la clase.
¿Qué pasa si escribís un método como public static void main sin encerrarlo dentro de una clase?	El archivo no compila. En Java no existen métodos ni funciones sueltas: todo método tiene que vivir dentro de la declaración de una clase, con la forma class NombreClase { }.
¿Qué pasa si escribís un while o un if directamente en el cuerpo de una clase, sin ponerlo dentro de un método?	Tampoco compila. El cuerpo de una clase solo admite declaraciones (variables, métodos, otras clases); las instrucciones ejecutables como while o if necesitan estar dentro del cuerpo de un método.
¿Cómo se traza (rastrea) a mano un bucle while con varias variables para predecir qué imprime un programa?	Se anota el valor de cada variable justo antes de entrar al bucle, y se lo actualiza línea por línea en cada vuelta, aplicando cada instrucción y cada if en el orden exacto del código — nunca se adivina el resultado sin pasar por cada vuelta una por una.
¿Qué imprime System.out.print(x + "" + y + " ") si x vale 4 e y vale 10 (ambos int)?	Imprime el texto "410 ": los enteros se convierten a texto y se pegan uno al lado del otro sin espacio entre ellos (el espacio final es literal, viene del " " del final). No es una suma: 4 + 10 daría 14, un resultado totalmente distinto.
¿Qué imprime System.out.println(1 + 2 + "3") en Java, y por qué no da lo mismo que System.out.println("1" + 2 + 3)?	El primero imprime "33": como 1 y 2 son ambos int, Java los suma primero (da 3) y recién después concatena con el String "3". El segundo imprime "123": al arrancar con un String, todos los + que siguen concatenan como texto en vez de sumar. El orden de los operandos cambia todo.
¿Por qué conviene usar print() en vez de println() cuando querés armar una palabra o frase juntando varios fragmentos de texto?	Porque print() no salta de línea: cada System.out.print() sucesivo se pega justo después del anterior, en la misma línea. Con println() cada fragmento quedaría en su propia línea, rompiendo la palabra que se quiere armar.
¿Qué pasa si un while nunca modifica la variable que usa su condición, por ejemplo while (x < 10) sin ningún x = x + 1 adentro?	El programa compila y corre sin errores, pero la condición nunca se vuelve false: queda en un bucle infinito, corriendo para siempre sin terminar.
¿Puede el código de un while o un if estar escrito directamente dentro de una clase, sin ponerlo dentro de un método?	No. Todo bucle o condicional necesita vivir dentro de un método; si queda suelto en el cuerpo de la clase, el programa no compila.
Al trazar un while con varios if seguidos (no if/else encadenados) dentro del bucle, ¿cuántos de esos if se pueden disparar en la misma vuelta?	Todos los que cumplan su condición. Al ser if independientes entre sí (no un if/else donde solo uno gana), cada uno se evalúa por separado, así que más de uno puede ejecutarse en la misma vuelta del bucle.
En el enfoque PROCEDURAL de programar, ¿en qué se piensa primero al diseñar un programa?	En los PROCEDIMIENTOS: qué acciones o funciones tiene que hacer el programa (por ejemplo rotate(), playSound()), sin partir de qué "cosas" del problema están involucradas.
En el enfoque ORIENTADO A OBJETOS (OO) de programar, ¿en qué se piensa primero al diseñar un programa?	En las COSAS (objetos) que existen en el problema (por ejemplo Square, Circle, Triangle), dándole a CADA UNA su propio comportamiento, en vez de una función genérica que decide qué hacer según un dato recibido.
Dos programadores reciben la misma consigna: varias formas geométricas que deben rotar y reproducir un sonido propio. Uno usa un enfoque procedural y el otro orientado a objetos. ¿Cómo resuelve cada uno el problema?	El procedural escribe un solo procedimiento genérico rotate(shapeNum) que recibe un número identificando la forma. El orientado a objetos escribe una clase por cada forma (Square, Circle, Triangle), cada una con su propio método rotate() que sabe rotar exactamente esa forma.
¿Qué es una "spec" (specification) en desarrollo de software?	El documento que describe QUÉ tiene que hacer el programa (el comportamiento esperado desde afuera), sin decir CÓMO programarlo por dentro.
Siguiendo el caso anterior, la consigna cambia: se agrega una forma nueva que necesita rotar de manera distinta a las demás. ¿Qué tuvo que hacer cada programador (procedural vs. orientado a objetos) para adaptarse a ese cambio?	El orientado a objetos solo modificó el método rotate() DENTRO de la clase nueva, sin tocar el código ya probado de las otras formas. El procedural tuvo que agregar parámetros nuevos a su procedimiento compartido rotate(), lo que afectó y rompió código que antes funcionaba, obligándolo a testear todo de nuevo.
¿Qué significa que el comportamiento y los datos de un objeto estén "encapsulados" dentro de su propia clase?	Que viven juntos y aislados dentro de esa clase: un cambio ahí adentro no obliga a tocar ni volver a probar el código de las demás clases del programa, a diferencia de un procedimiento genérico compartido por todos los casos.
¿Qué es un "attribute" (atributo) de una clase en POO?	Un dato propio que va a tener CADA objeto de esa clase (por ejemplo, la posición desde la que un objeto debe rotar) — es el nombre informal de lo que formalmente se llama "instance variable" (variable de instancia).
En POO, ¿qué es la HERENCIA (inheritance) entre clases?	Una relación donde una clase (subclase) hereda automáticamente los métodos y atributos de otra clase más general (superclase), sin tener que volver a escribirlos.
¿Qué es una SUPERCLASE (superclass) y qué es una SUBCLASE (subclass)?	La superclase es la clase más general/abstracta que define comportamiento común (ej. Shape con rotate() y playSound()). Las subclases son las clases más específicas que heredan ese comportamiento (ej. Square, Circle, Triangle, Amoeba heredan de Shape).
¿Qué significa hacer OVERRIDE (sobrescribir) de un método heredado en Java?	Que una subclase redefine (escribe su propia versión de) un método que heredó de su superclase, cuando necesita cambiar o extender ese comportamiento para su caso particular.
Si una subclase hereda un método pero necesita comportarse distinto a las demás subclases, ¿cómo lo logra, y cómo sabe el programa en tiempo de ejecución cuál versión usar?	La subclase hace OVERRIDE de ese método (lo redefine con su propio código). En tiempo de ejecución (runtime), la JVM decide automáticamente qué versión del método correr según el tipo real del objeto.
¿Qué gana un programa cuando el resto del código solo "invoca" (llama) un método como rotate() sobre un objeto, sin saber CÓMO lo hace por dentro?	Que se puede agregar un tipo de objeto nuevo (una clase nueva) sin tocar el resto del programa: cada objeto nuevo trae su propio comportamiento, y quien lo usa no necesita cambiar nada.
Al diseñar una clase en POO, ¿qué dos categorías básicas de información hay que pensar sobre los objetos que va a crear?	Las cosas que el objeto SABE (sus atributos, su estado) y las cosas que el objeto SABE HACER (sus métodos, su comportamiento).
¿Por qué el código que usa un objeto no necesita saber CÓMO hace ese objeto lo que le piden, solo QUÉ le están pidiendo?	Porque el objeto es responsable de implementar su propio comportamiento (el "cómo") de forma interna y aislada; quien lo usa solo invoca el método (el "qué") y confía en el resultado. Esto permite agregar objetos nuevos sin tocar el resto del programa.
Nombrá dos ventajas prácticas de la Programación Orientada a Objetos frente a la programación procedural, de las que valoran los programadores en su trabajo diario.	No hace falta tocar ni volver a testear código ya probado para agregar una funcionalidad nueva (alcanza con escribir una clase nueva), y los datos junto con los métodos que operan sobre ellos quedan juntos y organizados en una misma clase.
¿Qué es una "instance variable" (variable de instancia) en una clase Java?	Un dato que un objeto SABE sobre sí mismo (su estado); cada objeto de esa clase puede tener un valor distinto para la misma variable.
En POO, ¿qué significa que "instance" es otra forma de decir "object"?	Que un objeto ES una instancia de su clase: "instance" y "object" se usan como sinónimos. Por eso "instance variable" es simplemente "una variable propia de cada objeto".
¿Qué es un método getter y qué es un método setter? Dá un ejemplo.	Un getter LEE el valor de una instance variable (ej. getAlarmTime()) y un setter lo ESCRIBE/cambia (ej. setAlarmTime()). Es común que un objeto tenga un par getter/setter por cada dato importante que maneja.
¿Dónde se diseñan las instance variables y los métodos de un objeto: en el objeto mismo o en otro lugar?	Se diseñan como parte de la CLASE (el plano). Cada objeto creado a partir de esa clase tiene su propia copia de los valores de las instance variables, pero todos comparten las mismas definiciones de métodos.
